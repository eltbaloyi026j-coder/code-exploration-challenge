# Testing – Using AI to Help with Testing

## 📌 Overview

In this exercise, I explored how AI can assist in writing and understanding tests for a Node.js application.

---

## 🧪 What is Testing?

Testing is the process of checking if the code works correctly and behaves as expected.

It helps catch bugs early and ensures reliability.

---

## 🤖 How AI Helped

I used AI to:

* Understand what parts of the application should be tested
* Generate example test cases
* Explain expected outputs for different inputs

---

## 🧩 Example Test Cases

### 1. Create Task

* Input: valid title and description
* Expected: task is created and stored

### 2. Invalid Date

* Input: wrong date format
* Expected: error message shown

### 3. Update Task Status

* Input: valid task ID and status
* Expected: task status updates correctly

### 4. Delete Task

* Input: task ID
* Expected: task is removed from storage

---

## 🛠️ Tools Mentioned

* Jest (testing framework included in the project)

---

## 💡 What I Learned

* Testing ensures code works as expected
* AI can help generate test ideas quickly
* Good tests cover both valid and invalid cases

---

## 🚀 Reflection

Before this, I didn’t think much about testing. Now I understand that testing is an important part of development and helps improve code quality.
