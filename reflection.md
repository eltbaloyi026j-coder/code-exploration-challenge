# Code Exploration Challenge Reflection

## 1. How did the AI’s explanation change your understanding of the algorithm?
Before using AI, I was just seeing code as lines that do something but I didn’t fully understand the logic behind it. After using AI, I now understand that the algorithm works step-by-step by calculating a score for each task based on priority, due date, status and tags, then sorting tasks based on that score.

## 2. What aspects were still difficult to understand?
The part that was still a bit confusing was how different factors like due dates and tags are combined into one score and how small changes affect the final result. Also understanding date calculations took some time.

## 3. How would you explain this algorithm to another junior developer?
I would explain it like this:
The algorithm gives each task a score based on importance.
- Higher priority = higher score
- Tasks due soon = higher score
- Completed tasks = lower score
Then it sorts all tasks so the most important ones come first.

## 4. Did you test this understanding against AI?
Yes, I used AI to explain the algorithm and compared it with the actual code. I also ran the CLI project to see how tasks behave.

## 5. How might you improve the algorithm?
I would improve it by allowing custom scoring and making the code more readable by splitting logic into smaller functions.

Before this course, I couldn’t run or understand a full Node.js CLI project. Now I can run commands, debug issues, and understand how different files work together.
